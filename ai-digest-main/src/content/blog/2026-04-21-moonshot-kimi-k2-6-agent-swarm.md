21 апреля 2026 года китайский Moonshot AI перевёл в общедоступный релиз Kimi K2.6 — open-weight модель trillion-parameter MoE-архитектуры, доступную в бете с 13 апреля. Главный заявленный фокус — long-horizon autonomy: способность вести задачи часами без потери контекста и координировать большие группы агентов. Веса опубликованы на Hugging Face, доступ также открыт через Kimi.com, API и CLI Kimi Code.

## Четыре варианта и swarm на 300 агентов

K2.6 поставляется в четырёх конфигурациях: Instant (быстрые ответы), Thinking (расширенное рассуждение), Agent (одиночный исполнитель задач) и Agent Swarm — режим координации до 300 саб-агентов на горизонте до 4000 шагов. По заявленным замерам Moonshot, в режиме Agent модель ведёт до 13 часов непрерывного кодинга — например, рефакторинг крупного репозитория с прогоном тестов между правками. Контекст в 262 144 токена позволяет держать в памяти большие кодовые базы или собрания документов целиком, а нативная мультимодальность (текст, изображение, видео) убирает потребность в отдельных моделях для парсинга интерфейсов.

Swarm-режим — это не просто параллельный запуск инстансов. K2.6 включает протокол координации между саб-агентами с разделением ролей и общей памятью задачи. Это сближает её по сценариям с закрытыми системами вроде OpenAI Codex или Devin, но с принципиальной разницей: open weights позволяют запускать модель локально и встраивать в инфраструктуру без отправки данных провайдеру.

## Что это меняет для практиков

Появление serious-grade open-weight модели с агентным swarm-стеком меняет расклад для команд, которым важна приватность данных или предсказуемая стоимость. Closed-source провайдеры остаются впереди по продуктовой полировке, но разрыв в возможностях сокращается — особенно для задач, где можно поднять K2.6 на собственном железе. Для оценки имеет смысл начать с Agent-режима на изолированной кодовой задаче и сравнить с привычным стеком по метрикам времени и качества патча.
